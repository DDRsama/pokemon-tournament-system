### 叠加层模块化计划

- 将 `/t/<tournamentId>/overlay` 重构为 Overlay Shell + 独立 View 架构。
- `index.html` 作为外壳，保留 `#overlay-root`、`#overlay-buffer`、共享容器和 inert view templates。
- 不使用运行时 HTML fetch，不引入 Vue/React，不引入构建工具。
- 每个 View 拥有自己的 `init()`、`update()`、`destroy()` 生命周期。
- View 内副作用必须通过 `ViewContext` 管理。
- 定时器、事件监听、动画锁、自动滚动、彩带等副作用必须在 View 离开时销毁。
- View 切换使用双缓冲，避免 OBS 黑屏和布局闪烁。
- 同一 View 内的状态更新只调用 `update()`，禁止重挂载导致动画重播。

## Version 2.3

主题：模块化后的维护性整理与组件准备

### 主要目标

- 将模块化后的叠加层整理成更适合长期维护的代码结构。
- 抽取可复用叠加层组件。
- 为主题系统和语言系统做准备，但不改变现有行为。

### 计划内容

- 抽取选手名牌、比分块、赛段标签、logo 位置、连接线、结算徽章、二维码区块等共享组件。
- 将 CSS 拆为 base、components、各 view 专属样式。
- 减少重复选择器和状态特判。
- 为叠加层 class 增加更严格的命名约定。
- 增强状态路由测试和 view 生命周期检查。
- 文档化 OBS 安全渲染规则。

## Version 2.4

主题：主题系统、语言系统、赛事设置

### 主题系统

- 区分主题变量和结构主题。
- 主题变量覆盖颜色、阴影、边框、字体、logo 素材、透明度、动效时长。
- 结构主题允许改变组件布局和信息层级，而不仅仅是换色。
- 官方赛事风格比分牌应作为未来的结构主题，而不是当前社区赛事风格的简单换色。

### 语言系统

- 增加前端中英文日文文本字典。
- 赛事名和选手名保持原文，系统标签按语言切换。
- 让主页、后台、选手端、叠加层具备语言切换基础。

### 创建比赛设置

- 在创建比赛时加入结构化赛事设置。
- 计划包含瑞士轮 BO 数、是否开启淘汰赛、淘汰赛 BO 数、明牌/暗牌队伍模式，以及后续娱乐赛规则。
