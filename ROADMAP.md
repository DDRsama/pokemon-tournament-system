# Pokemon Tournament System 路线图

简体中文 | [English](./ROADMAP.en.md) | [日本語](./ROADMAP.ja.md)

## Version 2.2.0

状态：已发布

主题：前端 UI 美化、叠加层前端完全重绘、叠加层结构模块化

### 主要更新

- 重整主页视觉与比赛入口管理体验。
- 将后台页调整为更清晰的单场比赛管理界面。
- 统一页面标题、浏览器 favicon、顶部品牌栏和版本展示。
- 优化主页比赛操作、选手入口二维码、删除确认和重命名弹窗。
- 整理后台右侧的访问地址、叠加层、选手端和 OBS 设置模块。
- 改进 Base URL 校验流程，避免误填 `localhost` 或 `127.0.0.1` 作为对外地址。

### Overlay 更新

- 面向 OBS 与现场大屏，重新设计 1920x1080 叠加层视觉。
- 重绘瑞士轮总览、瑞士轮直播桌、瑞士轮结算、瑞士轮结束页。
- 重绘 Top 8 树状图、Top 8 直播桌、Top 8 结算和领奖台。
- 优化长选手名显示，减少中文和英文长名在关键画面中的截断。
- 增加瑞士轮总览自动滚动、排名动效和大规模选手列表展示能力。
- 优化 Top 8 晋级路线、胜者高亮、目标名片变化和领奖台信息层级。
- 保留透明背景设计，适配直播画面与现场大屏叠加使用。

### Overlay 架构

- 将 `/t/<tournamentId>/overlay` 重构为 Overlay Shell + View Registry + 生命周期管理结构。
- 使用模板和独立 View 管理各个叠加层画面。
- 引入双缓冲切换，降低 OBS 黑屏、闪烁和旧画面残留风险。
- View 内定时器、事件监听、自动滚动和动画副作用统一通过生命周期上下文清理。
- 同一 View 内的数据更新优先走 `update()`，避免无意义重挂载和动画重播。
- 保持现有外部路由、OBS 链接、后端 API 和 Docker 部署方式不变。

### 比赛流程修复

- 增加瑞士轮快照回退逻辑，并禁止第一轮前回退。
- 修复瑞士轮平局结算展示。
- 修复比赛结束后后台和主页状态仍显示淘汰赛的问题。
- 修复战报导出路径和接口跟随单场比赛路由的问题。
- 修复叠加层、选手端、后台端的比赛绑定和旧入口跳转。

## Version 2.2.5-dev.0

状态：源码开发快照，不创建 GitHub Release，不发布 Docker 镜像

主题：服务端工程重构与单元测试补强

### 计划内容

- 将服务端从单一大型入口拆分为 app、config、core、storage、reports、realtime 和 routes 模块。
- 保持 2.2.0 的用户可见行为、前端路由、API 路径和 JSON 数据格式不变。
- 增加 Node 原生 unit test，覆盖瑞士轮、排名、Top 8、存储、玩家视图和战报数据。
- 收紧低风险输入校验，包括安全 tournamentId、BO3 分数范围和 winner 合法性。
- 作为后续版本开发基础，仅上传源码与 tag，不发布 Docker Hub 镜像。
## Version 3.0

状态：已发布

主题：赛事引擎重制、联赛积分与长期选手身份

### 计划内容

- 将当前固定的“瑞士轮 + Top 8 单败”流程迁移为可配置赛事阶段模型。
- 新增 `schemaVersion: 3`、`tournamentSettings`、`stages`、`matchRules` 等底层数据结构。
- 保留当前默认流程作为内置规则模板，确保旧比赛和旧流程继续可用。
- 通用化 BO1 / BO3 / BO5 对局结果模型。
- 新增服务器端选手池，区分长期选手档案与单场报名记录。
- 支持临时选手参赛，但临时选手不进入联赛 / 赛季积分。
- 新增联赛 / 赛季、积分规则、计分赛和排行榜基础能力。
- 首页、后台、选手端、overlay 和战报逐步适配新的赛事引擎 view model。
- 为迁移、阶段推进、BO、选手档案、临时选手、联赛积分补充测试。

### 后续主题

- 主题系统。
- overlay 组件化与主题准备。
- 小组赛、双败淘汰、团队赛等扩展赛制的完整 UI。

## Version 3.1

状态：已发布

主题：体验、视觉和部署可靠性整理

### 计划内容

- 保留 `/player/` 独立选手中心入口，支持添加到手机主屏幕。
- 保留 Docker `/data` 单目录持久化，降低群晖和单容器部署成本。
- 精简选手中心排版，减少大按钮和大卡片，保留登录、报名、返回比赛和战报导出能力。
- 修正 Overlay 内部英文标签、TopN 文案、完赛领奖台优先级、比分视觉、动画裁切和标签溢出。
- 收尾后台小组赛展示、等待对手禁用视觉、Overlay 预览清晰度，以及主页档案管理列表密度。
- 更新 README 与发布说明，恢复三语 changelog。

### 不做内容

- 不做完整主题系统。
- 不做真实账号登录系统。
- 不新增核心赛制模型。
- 不重写 Overlay 架构。

## Version 3.x

主题：主题系统与扩展赛制

### 主题系统

- 区分主题变量和结构主题。
- 主题变量覆盖颜色、阴影、边框、字体、logo 素材、透明度和动效时长。
- 结构主题允许改变组件布局和信息层级，而不仅仅是换色。
- 官方赛事风格比分牌应作为未来结构主题，而不是当前社区赛事风格的简单换色。
