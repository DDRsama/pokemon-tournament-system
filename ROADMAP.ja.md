## Version 2.2.0

状態：リリース済み

テーマ：フロントエンド UI 改善、オーバーレイ全面再描画、オーバーレイ構造のモジュール化

### 主な更新

- ホーム画面と大会入口管理の体験を整理しました。
- 管理画面を単一大会向けの分かりやすいバックエンド画面に調整しました。
- ページタイトル、ブラウザー favicon、上部ブランドバー、バージョン表示を統一しました。
- ホーム画面の大会操作、プレイヤー入口 QR コード、削除確認、名前変更ダイアログを改善しました。
- 管理画面右側の Base URL、オーバーレイ、プレイヤーページ、OBS 設定まわりを整理しました。
- `localhost` や `127.0.0.1` を誤って外部公開 URL として使わないよう、Base URL 検証を改善しました。

### オーバーレイ更新

- OBS と会場スクリーン向けに、1920x1080 のオーバーレイを再設計しました。
- スイスドロー総覧、スイスドロー配信卓、スイスドロー結果演出、スイスドロー終了画面を描き直しました。
- Top 8 トーナメント表、Top 8 配信卓、Top 8 結果演出、表彰台画面を描き直しました。
- 中国語名と英語名の長いプレイヤー名表示を改善しました。
- スイスドロー総覧に自動スクロール、順位移動アニメーション、大規模参加者表示を追加しました。
- Top 8 の進出ライン、勝者ハイライト、進出先カード変化、表彰台の情報階層を改善しました。
- 配信画面と会場スクリーンに重ねやすい透明背景オーバーレイ設計を維持しています。

### オーバーレイ構造

- `/t/<tournamentId>/overlay` を Overlay Shell + View Registry + ライフサイクル管理構造へ再構築しました。
- テンプレートと独立 View によって各オーバーレイ画面を管理します。
- ダブルバッファ切り替えを導入し、OBS の黒画面、ちらつき、旧画面残りのリスクを軽減しました。
- タイマー、イベントリスナー、自動スクロール、アニメーション副作用は View ライフサイクルで整理されます。
- 同じ View 内の状態更新は `update()` を優先し、不要な再マウントや入場アニメーションの再生を避けます。
- 既存の外部ルート、OBS リンク、バックエンド API、Docker デプロイ方式は維持しています。

### 大会進行の修正

- スイスドローのスナップショット巻き戻し機能を追加し、第 1 ラウンド前の巻き戻しを禁止しました。
- スイスドローの引き分け結果表示を修正しました。
- 大会終了後も管理画面やホームで決勝トーナメント段階と表示される問題を修正しました。
- 単一大会ルート配下でのレポート出力パスと API を修正しました。
- オーバーレイ、プレイヤーページ、管理画面の大会バインドと旧入口リダイレクトを修正しました。

## Version 2.2.5-dev.0

状態：ソース開発スナップショット。GitHub Release は作成せず、Docker イメージも公開しません

テーマ：サーバー側のエンジニアリングリファクタと unit test 拡充

### 予定範囲

- 大きな単一サーバー入口を app、config、core、storage、reports、realtime、routes モジュールへ分割します。
- 2.2.0 のユーザー向け振る舞い、フロントエンドルート、API パス、JSON データ形式は維持します。
- Node.js 標準の unit test を追加し、スイスドロー、順位計算、Top 8、ストレージ、プレイヤービュー、レポートデータをカバーします。
- 安全な tournamentId、BO3 スコア範囲、合法的な勝者判定など、低リスクの入力検証を強化します。
- 後続開発の土台として、ソースコードと tag のみを残し、Docker Hub には公開しません。
## Version 2.3

テーマ：大会ルールと進行設定

### 予定内容

- 大会作成時に構造化された大会ルール設定を追加します。
- スイスドロー、グループステージ、トーナメントステージの設定を対象にします。
- BO 数設定と、シングルエリミネーション / ダブルエリミネーション形式をサポートします。
- 個人戦、チーム戦の基本フィールドを追加します。
- 管理画面、プレイヤーページ、オーバーレイ、レポートが同じルールデータを参照するようにします。
- 古い JSON データには既定値を補完し、互換性を維持します。
- ルール関連の進行処理に対するテストを追加し、進出、対局、レポートの安定性を確保します。

## Version 2.3.5-dev

テーマ：オーバーレイのコンポーネント化とテーマ準備

### 予定内容

- モジュール化されたオーバーレイ構造をさらに整理し、旧互換コードを減らします。
- プレイヤー名札、スコアブロック、フェーズラベル、logo エリア、接続線、結果バッジ、QR ブロックなどを共有コンポーネント化します。
- overlay CSS を base、components、view 専用スタイルへさらに分割します。
- 状態ルーティング、View ライフサイクル、OBS 安全レンダリングに関するチェックを強化します。
- 将来のテーマ作業に向けた構造を整えますが、この段階では完全なテーマ切り替えは導入しません。

## Version 2.4

テーマ：言語システムとテーマシステム

### 言語システム

- 中国語、英語、日本語の UI ラベル辞書を追加します。
- 大会名とプレイヤー名は原文のままにし、システムラベルのみ翻訳します。
- ホーム、管理画面、プレイヤーページ、オーバーレイを言語切り替えに対応させます。

### テーマシステム

- テーマトークンとレイアウトテーマを分離します。
- テーマトークンは色、影、枠線、タイポグラフィ、logo 素材、透明度、モーション時間を扱います。
- レイアウトテーマは色だけでなく、コンポーネント構造と情報階層も変更できるようにします。
- 公式大会風スコア表示は、現在のコミュニティ大会風 UI の単純な色替えではなく、将来のレイアウトテーマとして扱います。
