### オーバーレイモジュール化計画

- `/t/<tournamentId>/overlay` を Overlay Shell + 独立 View 構成に再構築する。
- `index.html` は shell として `#overlay-root`、`#overlay-buffer`、共通コンテナ、inert view templates を保持する。
- 実行時 HTML fetch、Vue/React、ビルドツールは使わない。
- 各 View は `init()`、`update()`、`destroy()` を持つ。
- View 内の副作用は `ViewContext` 経由で管理する。
- タイマー、イベントリスナー、アニメーションロック、自動スクロール、紙吹雪などは View 終了時に破棄する。
- View 切り替えはダブルバッファを使い、OBS の黒画面やレイアウトちらつきを避ける。
- 同じ View 内の状態更新では再マウントせず、`update()` のみを呼び出す。

## Version 2.3

テーマ：モジュール化後の整理とコンポーネント準備

### 主な目標

- モジュール化されたオーバーレイを、長期保守しやすいコードベースに整理する。
- 再利用可能なオーバーレイコンポーネントを抽出する。
- 動作を変えずに、テーマシステムと言語システムの準備を行う。

### 予定内容

- プレイヤー名札、スコアブロック、フェーズラベル、logo 配置、接続線、結果バッジ、QR ブロックなどの共有コンポーネント化。
- CSS を base、components、view-specific ファイルに分割。
- 重複セレクタと状態別の特例処理を削減。
- オーバーレイ class の命名規則を強化。
- 状態ルーティングテストと View ライフサイクルチェックの改善。
- OBS 安全レンダリング規則の文書化。

## Version 2.4

テーマ：テーマ、言語、作成時の大会設定

### テーマシステム

- テーマトークンとレイアウトテーマを分離する。
- テーマトークンは色、影、枠線、タイポグラフィ、logo 素材、透明度、モーション時間を扱う。
- レイアウトテーマは色だけでなく、コンポーネント構造も変更できるようにする。
- 公式大会風スコア表示は、現在のコミュニティ大会風 UI の単純な色替えではなく、将来のレイアウトテーマとして扱う。

### 言語システム

- 中国語、英語、日本語の UI ラベル辞書を追加する。
- 大会名とプレイヤー名は原文のままにし、システムラベルのみ翻訳する。
- ホーム、管理画面、プレイヤーページ、オーバーレイを言語切り替えに対応させる。

### 大会作成設定

- 大会作成時に構造化された設定項目を追加する。
- 予定項目には、スイスドロー BO 数、Top 8 の有効 / 無効、Top 8 BO 数、公開 / 非公開チームリストモード、将来のエンタメ大会形式などを含める。
